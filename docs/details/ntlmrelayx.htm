- ツール概要

カテゴリ
通信の不正中継
説明
NTLM認証を窃取する中間サーバーとして動作し、被害者になりすまして、対象サーバへの認証を取得する。
攻撃時における想定利用例
被害者の認証情報を用いた、認証サーバーへの侵入

- 実行成功時に確認できる痕跡

  • 接続元
    • ntlmrelayx.py の対話モード（-i）で SMBのNTLMリレーが成功した際、「セキュリティ」ログ イベント5154・5158の「送信元アドレス」に「127.0.0.1」が記録される。
  • 接続先
    • NTLMリレー成功時に、「セキュリティ」ログのログイン成功（イベント4624）の「ソース ネットワーク アドレス」にNTLMリレーサーバーのIPアドレスが記録される。

- 備考

  • 本ツールの動作確認においては、WindowsのLanmanServerの無効やツール側（ntlmrelayx.py）のオプション（--no-server）により、Windowsとツールが利用するポートの競合によるツールの実行エラーを回避している
  • 本ツールの動作確認においては、SMBのNTLM Relayを発生させて確認している。SMB証明が有効である場合、SMBのNTLM Relayは失敗する
  • 本ツールの動作確認においては、リレーされるNTLM認証自体は別端末で発生させている
